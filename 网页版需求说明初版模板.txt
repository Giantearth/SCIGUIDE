脊髓损伤治疗参考网页需求说明

1. 项目目标
项目名称：
使用对象：治疗师 / 康复治疗师 / 实习治疗师
核心用途：基于 `DATABASE` 里的 Markdown 内容，做一个只读型参考网页
使用场景：门诊查阅 / 教学培训 / 居家指导 / 其他
项目定位：以“快速查看推荐内容 + 自行展开阅读完整指南”为主，不追求精确检索

2. 内容范围
当前内容来源：`DATABASE` 内全部 Markdown 文件
是否允许使用数据库外新增内容：不允许
是否允许后续补充图片：允许 / 不允许
是否允许后续补充 PDF / 表格 / 视频：允许
是否允许把四国指南原文或附件挂到网页中供用户自行阅读：允许
内容更新方式：我继续在 Obsidian 更新，再同步

3. 内容规则
所有展示内容必须仅来源于 `DATABASE` 和四国指南原文附件
不允许模型自由生成医学结论
是否允许做以下处理：
Markdown 排版渲染：允许
双链跳转解析：可选
标签聚合：允许
标题目录生成：允许
关键词搜索：不需要
对原文做轻度结构化拆分：允许
折叠显示非核心内容：允许
按条件做粗筛选：允许

4. 核心使用方式
首页录入信息后，根据录入的病因、节段、ASIA分级、病程时长和职业分类，给出相应内容页面
不做搜索框，不做精确关键词检索
重点做“推荐阅读 + 完整内容展开 + 原始指南挂载阅读”

5. 首页录入信息
首页主标题：患者信息录入
首页说明语：录入患者基本情况后，系统展示优先参考内容，其余相关内容可展开查看

首页字段：
患者编号输入：可选数字或者字母缩写，不允许输入汉字（保护隐私）
患者病因输入：创伤性 / 非创伤性
非创伤性下级分类：根据 DATABASE 文件里的分类进行预填写供用户选择
患者损伤节段输入：给予可选项（颈段：C1-C7，胸段：T1-T12，腰骶段：L1-S5）
ASIA分级输入：A / B / C / D / E
患者病程时长选择：72h内 / 14天内 / 14天以上
用户职业选择：PT / OT / ST / 呼吸康复

6. 结果页逻辑
结果页根据首页信息直接进入筛选后的相应内容页
红旗征为优先显示，且默认不折叠
其余内容按模块分组显示，可折叠或选择是否展开

建议显示顺序：
红旗征
康复管理
康复目标
康复重点
康复预期
痉挛管理
疼痛管理
转移训练
居家指导
完整原文入口
四国指南原文入口

7. 分类方式
是否优先沿用现有 MD 的结构：是
是否要把以下内容单独做快捷入口：是
康复红旗征
病程分期
ASIA / AIS
节段分类
并发症管理
功能训练指南
四国指南原文

8. 职业分类策略
职业分类只做大概筛选，不要求非常精细
PT / OT / ST / 呼吸康复对应优先显示相关内容
若同一内容与多个职业相关，可重复归类或同时显示
不能明确分类的内容放入“其他相关内容”折叠区

9. 页面结构
首页：录入信息
结果页：根据条件展示推荐内容
分类页：直接浏览全部分类内容
文章详情页：展示单篇内容
指南原文页：展示四国指南原文或附件入口
收藏页（可选）
关于页（可选）

10. 文章详情页要求
是否显示原始标题：是 / 否
是否显示来源：是
是否显示标签：是
是否显示面包屑导航：否
是否显示文内目录：否 / 可选
是否支持点击双链跳转：否 / 可选
是否支持复制文本：是
是否需要“相关推荐”：否
是否提供“查看原始指南出处”入口：是

11. 搜索要求
不设置搜索页
不做关键词检索
如后续需要，只保留最简单的站内标题筛选作为候选功能

12. 视觉和交互
风格倾向：专业医疗
主色倾向：白色，浅蓝色
是否需要深色模式：需要
字体大小偏好：中
是否更强调快速查阅而非视觉设计：是
是否允许大量折叠面板：允许
是否优先适配手机端：是

13. 技术限制
是否必须离线可用：否

是否需要登录：否

是否需要数据库：否
